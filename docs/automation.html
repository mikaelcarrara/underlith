AI & Automation Integration

Underlith can act as a stable design layer in AI-assisted and automated workflows. Its explicit token structure allows automation tools and AI systems to read, validate, and generate UI code without coupling to any specific framework.

AI-assisted workflows

Design tokens in Underlith are explicit, semantic, and framework-agnostic. This makes them suitable as inputs and constraints for automation and AI systems.

Rather than generating arbitrary styles, AI operates within a bounded design space defined by tokens.

Requirements for Automation

To enable the use cases below, you must first generate the machine-readable tokens.json file. This file acts as the bridge between your CSS variables and external tools (linters, AI agents, CI/CD). 

# 1. Install dependencies
npm install

# 2. Build tokens.json
npm run build:tokens

# Output: ./tokens.json

Real-world Use Cases to Implement Today

1. Pre-commit Design Linting

# .husky/pre-commit
#!/bin/bash
Underlith-lint src/**/*.css --tokens=./tokens.json
# Blocks commits with hardcoded values

2. AI-Assisted Component Library Migration

# Incremental migration
$ ai migrate ./legacy-components \
  --from chakra-ui \
  --to Underlith \
  --preserve-behavior
# Generates diffs for human review

3. Design QA in CI/CD

# .github/workflows/design-qa.yml
- name: Validate Design Tokens
  run: |
    ai validate-pr \
      --base main \
      --head ${{ github.head_ref }} \
      --tokens Underlith
    
    # Comments on PR:
    # "⚠️ 3 new hardcoded colors found"

4. AI Code Review Bot

// PR review bot
export async function reviewPR(diff: string) {
  const prompt = `
    Analyze this CSS diff.
    Report violations of Underlith tokens.
    Suggest token-compliant alternatives.
    
    Allowed tokens: ${Underlith_TOKENS}
  `;

  const review = await llm.analyze(prompt, diff);
  await github.comment(review);
}

Roadmap: Future Integration

2. JSON Schema & TypeScript Types

// Should have autocomplete
import { UnderlithTokens } from '@Underlith/tokens';

type SpacingToken = UnderlithTokens['spacing']; // '1' | '2' | '3' | ...
type ColorToken = UnderlithTokens['colors']['primary']; // '100' | '200' | ...

// Build-time validation
const Button = styled.button<{ spacing: SpacingToken }>`
  padding: var(--ul-space-${props => props.spacing});
`;

4. AI Context Injection Helper

// For use with Cursor, Copilot, etc.
import { injectUnderlithContext } from '@Underlith/ai';

// Automatically adds tokens to the context window
const contextFile = injectUnderlithContext('./tokens.json');
// → Generates .cursorrules, .github/copilot-instructions.md
