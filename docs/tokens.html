Tokens

Tokens are the foundation layer of Underlith.

They represent design decisions expressed as variables, not components.
In Underlith, a token is a CSS Custom Property, such as: 

--space-1
--font-size-md
--color-neutral-900
--radius-md

They define constraints, scales, and semantic meaning that other layers consume.

What tokens are (and are not)

Tokens are:

  • Named CSS variables (--space-1, --color-brand-500)
  • Explicit design decisions
  • Stable and versionable contracts
  • Independent from layout and components

Tokens are not:

  • UI elements
  • Styled components
  • Visual examples
  • Framework-specific utilities

If something has a default appearance, it belongs to Components, not here.

Token structure

Each file exposes a set of CSS variables focused on a single decision domain.

Typography typography.css

Defines typographic scales as tokens.

Example

--font-size-sm
--font-size-md
--font-size-lg
--line-height-tight
--font-weight-regular

These tokens describe typographic intent, not headings or text styles.

Spacing spacing.css

Defines the spacing scale used across the system.

Example

--space-1
--space-2
--space-3
--space-4
--space-5

Spacing tokens replace arbitrary values like 8px or 12px with a consistent scale.

Colors colors.css

Defines raw color values.

Examples

--color-brand-300
--color-brand-500
--color-neutral-0
--color-neutral-900

These tokens are value-based, without semantic context.

Radius and borders radius-and-borders.css

Defines shape-related decisions.

Examples

--radius-sm
--radius-md
--radius-lg
--border-width-hairline

Used to ensure consistent geometry across all consumers.

Elevation elevation.css

Defines depth tokens.

Examples

--elevation-1
--elevation-2
--elevation-3

Elevation tokens express relative depth, not specific components.

Motion motion.css

Motion tokens: durations, easings, delays and composite tokens; reduced-motion policy.

Durations

--duration-nano
--duration-micro
--duration-fast
--duration-base
--duration-moderate
--duration-slow
--duration-glacial
--duration-epic

Easing

--ease-linear
--ease-enter
--ease-enter-back
--ease-enter-spring
--ease-exit
--ease-exit-sharp
--ease-inout
--ease-inout-soft
--ease-inout-strong
--ease-bounce
--ease-bounce-strong
--ease-elastic
--ease-anticipate
--ease-sine-in
--ease-sine-out
--ease-sine-inout
--ease-breath
--ease-float
--ease-drift

Delays

--delay-none
--delay-xs
--delay-sm
--delay-md
--delay-lg
--delay-xl
--delay-2xl
--delay-3xl
--stagger-unit

Composite motion tokens

--motion-drop-bounce
--motion-fade-up
--motion-fade-in
--motion-scale-pop
--motion-glow-breath
--motion-pulse
--motion-particle
--motion-slide-left
--motion-slide-right
--motion-float
--motion-shimmer
--motion-skeleton
--motion-skeleton-fill
--motion-flip-in

Use durations/easings for fine control, or the composite tokens for common effects.

Opacity opacity.css

Defines transparency levels.

Examples

--opacity-disabled
--opacity-muted
--opacity-overlay

Opacity tokens avoid arbitrary alpha values.

Breakpoints breakpoints.css

Defines responsive thresholds as tokens.

Examples

--breakpoint-sm
--breakpoint-md
--breakpoint-lg

Breakpoints define when layout decisions change.

Semantic aliases semantic-aliases.css

Maps raw tokens to semantic meaning.

Examples

--color-text-primary
--color-text-secondary
--color-background-surface
--color-border-subtle

Semantic aliases allow design intent to evolve without changing the underlying scale.
