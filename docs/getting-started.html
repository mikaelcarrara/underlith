Getting started

Underlith is the infrastructure for your own source of truth.
Two commands for new projects. Three for existing ones.

Install

Add Underlith to your project via npm:

npm install @mikaelcarrara/underlith
New project

Two commands. Done.

Define your brand tokens, scaffold the package, publish. Every product your org builds from here installs @your-org/tokens and inherits your design decisions automatically.

$ underlith brand init → $ npm publish

1. Scaffold your brand package

Run the command and follow the prompts. Underlith will ask for your org name and generate a ready-to-publish package with your brand tokens separated from framework primitives.

underlith brand init --org acme

This generates:

@acme/tokens/
├── package.json
├── README.md
├── acme.brand.css   ← your brand tokens
└── acme.base.css    ← framework primitives

2. Publish to npm

cd @acme/tokens
npm publish --access public

3. Install in any project

From this point, every product your org builds just installs the package:

npm install @acme/tokens
@import "@acme/tokens/acme.brand.css";

Change once in the package. Every product updates on the next npm update.

Existing project

Three commands. Zero rewrites.

Already have a design system? Map your existing variables to Underlith tokens, extract your brand layer, publish. Components keep working exactly as before.

$ underlith init --shadcn → $ underlith brand init → $ npm publish

1. Map existing variables

Point the script at your globals.css. It maps every CSS variable to an Underlith token without touching a single component.

underlith init --shadcn \
  --globals ./styles/globals.css

Before and after — zero component changes:

/* Before */
--primary: oklch(0.205 0 0);

/* After */
--primary: var(--ul-primary);

2. Extract your brand layer

Underlith separates your brand values from framework primitives and scaffolds a standalone package.

underlith brand init --org acme

3. Publish

cd @acme/tokens
npm publish --access public

From here, any project installs @acme/tokens and inherits the same brand decisions automatically.

Just consuming tokens?

If you only need to consume Underlith tokens directly — no brand package needed — import and use:

@import "@mikaelcarrara/underlith/src/underlith.tokens.css";

color: var(--ul-color-text);
gap: var(--ul-space-4);
border-radius: var(--ul-radius-md);

Works with plain CSS, Tailwind, Sass, or any CSS-in-JS solution. No framework assumptions.
