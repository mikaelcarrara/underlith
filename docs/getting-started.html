Getting started

Underlith is the infrastructure for your own source of truth.
Two commands for new projects. Three for existing ones.

Prerequisites

Node.js 18+ and an account at npmjs.com. You'll need your npm username or org name to publish your brand package.

Step 1 — Install the CLI globally

Install Underlith globally to make the underlith command available in any terminal session. Run this once.

npm install -g @mikaelcarrara/underlith

Step 2 — Create a project and install the package

Create a folder, initialize it, and install Underlith locally. The local install gives you access to the starter tokens file.

mkdir my-project
cd my-project
npm init -y
npm install @mikaelcarrara/underlith

Step 3 — Copy the starter tokens file

Copy the base token file to your project root. This is the file Underlith reads to scaffold your brand package.

cp node_modules/@mikaelcarrara/underlith/src/underlith.tokens.css .

Open underlith.tokens.css and edit the brand token values to match your org's colors and identity before the next step.

New project

Step 4 — Scaffold your brand package

Run brand init with your npm username or org name. Underlith reads your tokens file, separates brand tokens from framework primitives, and generates a ready-to-publish package. 

underlith brand init --org your-npm-username

Follow the prompts to name your package. When complete, the command shows the exact output path:

Next steps:
  1. cd ./@your-npm-username/your-package-name
  2. Review your-npm-username.brand.css
  3. npm publish --access public

This generates:

@your-npm-username/your-package-name/
├── package.json
├── README.md
├── your-npm-username.brand.css   ← your brand tokens
└── your-npm-username.base.css    ← framework primitives

Step 5 — Review and publish

Use the exact path the command printed — don't guess it. The folder name reflects your org and package name.

# Use the path from the output above — e.g.:
cd ./@your-npm-username/your-package-name

# Review your brand tokens before publishing
cat your-npm-username.brand.css

# Publish
npm publish --access public

Step 6 — Consume in any project

From this point, every product your org builds installs the package and inherits your brand decisions automatically.

npm install @your-npm-username/your-package-name
@import "@your-npm-username/your-package-name/your-npm-username.brand.css";

color: var(--ul-color-brand-primary);
gap: var(--ul-space-4);
border-radius: var(--ul-radius-md);

Change a token value in the package, publish a new version, and every product updates on the next npm update.

Existing project

Already have a design system?

Map your existing variables to Underlith tokens, extract your brand layer, publish. Components keep working exactly as before — zero rewrites.

$ underlith init --shadcn → $ underlith brand init → $ npm publish

Step 1 — Install the CLI globally

Run this once if you haven't already.

npm install -g @mikaelcarrara/underlith

Step 2 — Install the package locally

Install Underlith inside your existing project.

npm install @mikaelcarrara/underlith

Step 3 — Map existing variables

Point the script at your globals.css. It maps every CSS variable to an Underlith token without touching a single component.

underlith init --shadcn \
  --globals ./styles/globals.css

Before and after — zero component changes:

/* Before */
--primary: oklch(0.205 0 0);

/* After */
--primary: var(--ul-primary);

Step 4 — Scaffold your brand package

Use your npm username or org name. Underlith reads your tokens, separates brand from framework primitives, and generates a ready-to-publish package.

underlith brand init --org your-npm-username

The command shows the exact output path when complete:

Next steps:
  1. cd ./@your-npm-username/your-package-name
  2. Review your-npm-username.brand.css
  3. npm publish --access public

Step 5 — Review and publish

# Review your brand tokens before publishing
cat your-npm-username.brand.css

# Publish
npm publish --access public

Just consuming tokens?

If you only need to consume Underlith tokens directly — no brand package needed — import and use:

@import "@mikaelcarrara/underlith/src/underlith.tokens.css";

color: var(--ul-color-text);
gap: var(--ul-space-4);
border-radius: var(--ul-radius-md);

Works with plain CSS, Tailwind, Sass, or any CSS-in-JS solution. No framework assumptions.
