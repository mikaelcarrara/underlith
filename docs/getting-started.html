Getting started

Underlith is designed to be consumed, not imposed.
You can adopt it incrementally, partially, or as a full design foundation.

Installation

The recommended way to start a new Underlith project is via the CLI.

1. Via CLI (Recommended)

# Initialize configuration
npx Underlith init

# Generate tokens
npm run build:tokens

2. Manual Installation

Clone or download from GitHub:

git clone https://github.com/mikaelcarrara/Underlith.git

Or download directly and include in your project.

Import everything

@import "path/to/Underlith/src/Underlith.css";

This includes tokens, base styles, and reference components.

Or import only what you need

@import "path/to/Underlith/src/tokens.css";
@import "path/to/Underlith/src/base.css";

Using with CSS-in-JS

Underlith acts purely as a token layer.

Import tokens globally

import "path/to/Underlith/src/tokens.css";

Use tokens inside components

const Button = styled.button`
  padding: var(--space-3) var(--space-5);
  background: var(--brand-500);
  color: var(--neutral-0);
  border-radius: var(--radius-md);
`;

No coupling.
No runtime dependency.
No framework assumptions.
